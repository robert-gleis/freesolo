# Automated Pull Request Creation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `PullRequestCreator` to open GitHub PRs from verified candidate branches with artifact-driven bodies and provenance at `pull-request.json`.

**Architecture:** Extend `src/integration/` with PR types, Zod-validated provenance store, markdown body builder (summary from spec/plan, test table from `VerificationRun`, review embed from `findIssueArtifacts`), and `createPullRequest` with injectable `runGh`/`runGit`. Thin `issueflow pr create|show` CLI in `src/commands/pr.ts`. Preconditions enforce candidate-ready + verification-pass + review artifact present.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), Zod, execa, vitest, Commander.
