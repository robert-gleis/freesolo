> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist IssueFlow worktree metadata in `~/.issueflow/state.db` (SQLite, WAL), expose list/drift CLI commands, and upsert metadata from `issueflow start`.

**Architecture:** Add a `src/state/` module with path resolution (env override for tests), `better-sqlite3` bootstrap, a numbered SQL migration runner, and a `WorktreeStore` with upsert/list/drift helpers. Wire `start.ts` to upsert after worktree resolution and before session writes. Register `issueflow worktrees list|drift` on the CLI.

**Tech Stack:** TypeScript, Node.js 20 (ESM), better-sqlite3, Zod (errors only if needed), Commander, Execa, Vitest.

**Native dependency note:** `better-sqlite3` compiles native bindings via `node-gyp` on install. Contributors need platform build tools (Python, make; Xcode CLT on macOS; MSVC on Windows). CI must run `npm ci` successfully with native compilation — verify in Task 9 after adding the dependency.

```ts
import fs from 'node:fs';
import Database from 'better-sqlite3';

import { resolveStateDbPath, resolveStateDir } from './paths.js';

export class StateDbError extends Error {
  constructor(message: string, options?: { cause?: unknown }) {
    super(message, options);
    this.name = 'StateDbError';
  }
}

export function openStateDb(): Database.Database {
  const stateDir = resolveStateDir();
  try {
    fs.mkdirSync(stateDir, { recursive: true });
    const db = new Database(resolveStateDbPath());
    db.pragma('journal_mode = WAL');
    db.pragma('busy_timeout = 5000');
    return db;
  } catch (error) {
    throw new StateDbError('Failed to open IssueFlow state database', { cause: error });
  }
}
```
