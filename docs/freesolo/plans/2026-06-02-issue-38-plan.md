# Pi Agent Adapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `PiAgentAdapter` — a real `AgentAdapter` backed by `pi --mode rpc --offline --no-session` with injectable transport, log snapshots for the event log, and full unit tests (no live Pi in CI).

**Architecture:** Split JSONL/RPC plumbing (`pi-rpc.ts`) from the adapter lifecycle (`pi.ts`). Tests drive a fake transport via `createInMemoryPiTransport` and `transportFactory`. The workflow engine is untouched.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), `node:child_process`, vitest, existing `execa` only if needed for spawn in production path.

**Spec:** `docs/freesolo/specs/2026-06-02-issue-38-design.md`

---

## Acceptance Criteria Mapping

| Issue criterion | Plan tasks |
|-----------------|------------|
| Pi agents can be spawned via the adapter | Task 3 (`start()` + `transportFactory` spawn capture); Task 4 (`createNodePiTransport` / default constructor) |
| Adapter tracks agent status (idle / running / finished / errored) | Task 3 lifecycle tests: `idle`, `starting`, `running`, `stopping`, `stopped`, process-exit→`error`, `start-failed` |
| Adapter captures logs consumable by the event log | Task 1 (`AgentLogSnapshot` fields); Task 3 Steps 14–15 (`readLogs()` ring buffer + truncation flag) |
| Injectable transport / no live Pi in CI | Task 2 (`createInMemoryPiTransport`); Task 3 (`transportFactory`); Task 4 keeps tests on fake factory |
| JSONL framing (`\n` only, no readline) | Task 2 Steps 1–3 |
| Headless extension UI auto-cancel | Task 2 Steps 4–5 |
| `initialInstructions` via first `prompt`, not CLI flag | Task 3 Step 5 |
| Engine isolation (no `src/workflow/` imports of pi modules) | Task 5 Step 1b (`pi-engine-isolation.test.ts`) + Step 1 `rg` check |
