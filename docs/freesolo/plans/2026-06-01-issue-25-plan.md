# Tmux Runner Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `TmuxRunner` implementing the `Runner` interface with injectable tmux command execution, full unit coverage, and a repo-wide guard that tmux runtime usage exists only under allowed paths.

**Architecture:** Add `src/runners/tmux-command.ts` for session-name sanitization, default `execa('tmux', …)` execution (`runTmux`, `defaultTmuxRunnerDeps`), and `src/runners/tmux.ts` for `TmuxRunner` state machine logic. `TmuxRunnerDeps` includes `runTmux` and `now` (mirrors `verification-runner.test.ts`). Export from `src/runners/index.ts`. Unit tests mock `runTmux`; no workflow or CLI wiring in this ticket.

**Tech Stack:** TypeScript (strict, ES2022, NodeNext), execa (existing), vitest.
