# Reviewer Artifact Generation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automatically generate stable `TEST_REPORT.md` and `REVIEW_REPORT.md` artifacts after verification and review gates pass, persisted under `.git/freesolo/reports/issue-<N>/` and retrievable by issue id.

**Architecture:** New `src/reports/` module builds markdown from `VerificationRun` and session/review-round files, writes via `git rev-parse --git-path`. `createVerifyPlan` invokes an injected `writeTestReport` dep when returning `mode: 'completed'` (testable via `VerifyPlanDeps`, matching #20). `review-loop.mjs` invokes `write-review-report.mjs` after `record-review --status pass`. Session `artifacts.testReport` / `artifacts.reviewReport` updated on write via `updateSessionReportArtifact(worktreePath, …)`. `freesolo reports show` exposes retrieval.

**Tech Stack:** TypeScript (ESM, NodeNext), Zod (session schema only), Commander, Vitest, plain Node `.mjs` script bridge.

async function gitFreesoloPath(repoRoot: string, ...segments: string[]): Promise<string> {
  const joined = ['freesolo', ...segments].join('/');
  const { stdout } = await execa('git', ['rev-parse', '--git-path', joined], { cwd: repoRoot });
  const resolved = stdout.trim();
  return path.isAbsolute(resolved) ? resolved : path.join(repoRoot, resolved);
}

Default implementation:

```ts
writeTestReport: async (run) => {
  try {
    const reportPath = await writeTestReportToDisk(run);
    await updateSessionReportArtifact(run.repoRoot, 'testReport', reportPath);
    return reportPath;
  } catch (error) {
    console.error(`freesolo: failed to write TEST_REPORT.md: ${error instanceof Error ? error.message : String(error)}`);
    return null;
  }
}
```

Invoke inside `createVerifyPlan` after `deps.runPipeline` resolves and before returning `{ mode: 'completed', run, exitCode }`. `verifyAction` unchanged except it benefits automatically.
